# unified-agnostic-api

A shell-first, platform-agnostic API monorepo. Exposes any backend — filesystem,
GitHub, browser, OS, AI — through a unified HTTP + CLI interface built entirely
from bash, with Go-backed HTTP transport where needed.

## Architecture

```
unified-agnostic-api/
├── server/          # HTTP API server (shell2http + webhook + bash-api-server patterns)
├── cli/             # CLI layer generated by bashly
├── adapters/        # Backend adapters — one directory per domain
│   ├── filesystem/  # Virtual FS: zen-fs patterns, FUSE bridges, archive mounts
│   ├── github/      # GitHub REST API: org admin, releases, scripts
│   ├── browser/     # Browser automation: CDP, puppeteer, chromium
│   ├── os-compat/   # Cross-platform OS: file, process, clipboard, system info
│   └── ai/          # AI/LLM: agentic shell, model calls, ai-bom scanning
├── lib/             # Shared bash includes (sourced by all adapters + server)
├── tools/           # Dev/CI utilities (ai-bom, linting, validation)
├── config/          # Server config, route definitions, adapter manifests
└── docs/            # Architecture docs, adapter authoring guide
```

## Design principles

- **Shell-first**: every adapter is a bash script. The HTTP layer (`server/`) wraps
  them as endpoints via `shell2http` or `webhook`. No Go/Node/Python required at
  runtime unless an upstream dependency needs it.
- **Agnostic**: adapters are swappable. The server routes requests to whichever
  adapter is configured — local filesystem, remote API, browser, or AI model.
- **Composable**: adapters can be piped together. A GitHub adapter can feed a
  filesystem adapter which feeds an AI adapter.
- **Fork-sync-all native**: this repo is managed by
  [fork-sync-all](https://github.com/Interested-Deving-1896/fork-sync-all) and
  mirrored to OpenOS-Project-OSP. Consumer repos receive the `server/` and
  `adapters/` layers via the template system.

## Quick start

```bash
# Start the API server on port 8080
./server/start.sh --port 8080

# Use the CLI
./cli/uaa --help
uaa filesystem ls /tmp
uaa github repos list my-org
uaa ai complete "list all files modified today"

# Add a new adapter
cp adapters/filesystem/template.sh adapters/my-adapter/my-endpoint.sh
# Register it in config/routes.yml
```

## Adapter authoring

Each adapter is a directory under `adapters/` containing:
- `*.sh` — endpoint scripts (stdout = response body, exit 0 = 200, exit 1 = 500)
- `manifest.yml` — declares routes, methods, auth requirements
- `README.md` — adapter-specific docs

See `docs/adapter-authoring.md` for the full contract.
